# MineVOICE 产品 TODO

本清单按用户体验和发布风险排序。`已完成` 表示已有代码与本地验证路径；未完成项在合入前应满足写明的验收条件。

## P0: Alpha 联机稳定性

- [ ] **双客户端回归脚本**：一条命令启动语音服务器、Minecraft 服务端和两个客户端；验收为两名真实客户端可完成连接、范围语音、队伍语音、静音和重连检查，并保留可读日志。
- [ ] **设备热插拔恢复**：耳机或麦克风拔插后，系统默认设备变化时不应永久失声；验收为设置页能刷新设备列表，测试输入/输出能正确指向新设备。
- [ ] **连接状态与错误提示**：区分未配置、连接中、已连接、鉴权失败、UDP 超时和服务端断开；验收为 HUD 与设置页展示相同状态，日志包含可定位原因。
- [ ] **范围路由回归**：验收为同维度、跨维度、边界距离、静音、屏蔽听音、玩家个人音量均有自动化或双客户端测试覆盖。
- [ ] **队伍语音权限与生命周期**：验收为创建、加入、离开、队长离线、玩家退出服务器后都不会残留幽灵成员或继续转发音频。
- [ ] **Windows 测试运行器修复**：解决 Unicode 工作目录下 Gradle/JUnit 测试 worker 找不到测试类的问题；验收为 Windows 与 Linux 都执行完整单元测试，不能依赖跳过测试。

## P1: 音频质量与网络适应

- [ ] **Opus 编码与解码**：替换当前验证用 PCM 直通；验收为 48 kHz 单声道语音在目标码率下稳定收发，并有编码失败回退和版本协商策略。
- [ ] **抖动缓冲与丢包处理**：为 UDP 接收端加入有界队列、乱序处理和丢包隐藏；验收为人工注入乱序和轻度丢包时播放不断裂、延迟可配置且不无限增长。
- [ ] **语音激活检测**：实现输入电平检测、阈值和滞回；验收为阈值可在游戏内保存和调节，静音时不发送无效音频帧。
- [ ] **降噪与回声消除接口落地**：接入可替换 DSP 实现；验收为开关状态可持久化，处理耗时和异常不会阻塞游戏线程。
- [ ] **音量控制完善**：分别保存总音量、语音音量、麦克风增益和玩家个人音量；验收为重启后配置保留，并对正在播放的声音立即生效。

## P2: 空间语音与方块上下文

- [ ] **玩家位置同步稳定化**：以服务端权威的位置、维度和朝向驱动空间计算；验收为传送、换维、死亡重生和快速移动不会把声音路由给错误目标。
- [ ] **方块样本传输契约**：定义客户端和算法模块使用的方块材质、碰撞和遮挡采样格式；验收为协议版本化、数据大小受限，且队伍语音与范围语音的规则明确分离。
- [ ] **遮挡与衰减算法**：从简单射线和材质系数开始；验收为墙体、开口和不同距离产生可解释的音量/滤波变化，算法异常时回退到距离衰减。
- [ ] **立体声空间化完善**：在当前左右声道方向感基础上加入平滑、距离衰减和玩家朝向；验收为说话者移动时声像连续，不出现左右跳变或爆音。

## P3: 服务端运维

- [ ] **NeoForge 配置规范化**：迁移到 NeoForge config spec，并提供迁移说明；验收为服务端配置校验、默认值、热更新边界和错误提示明确。
- [ ] **管理员命令和诊断**：增加在线会话、队伍、丢包、带宽和连接状态查询；验收为命令不会暴露 token 或 `sharedSecret`。
- [ ] **独立服务器限流与容量保护**：增加每会话、每 IP 和全局带宽限制；验收为异常客户端不能拖垮 UDP 线程，指标可观测。
- [ ] **Docker 生产示例**：补齐安全的 secret 注入、UDP 端口、日志轮转和升级步骤；验收为示例不包含真实密钥且可在干净主机启动。

## P4: 发布与兼容性

- [x] **自动发布**：推送 `v*` 标签会构建 NeoForge 模组、独立语音服务器 zip、Docker 镜像和 GitHub Pre-release。
- [x] **部署包命名**：Release 明确区分模组 jar 与独立语音服务器 zip，并说明模组 jar 需要同时安装在客户端和 Minecraft 服务端。
- [ ] **升级兼容性策略**：定义配置、网络包和服务端版本的兼容矩阵；验收为不兼容组合会在连接前给出明确提示。
- [ ] **发布验收记录**：每个标签记录 Minecraft/NeoForge 版本、双客户端联机结果、已知问题和回滚版本。

## 不在当前 Alpha 范围内

- 内置 Local 语音服务器。
- 跨服语音网络。
- 录音、回放或服务器保存玩家音频。
- 面向大规模正式服的带宽与抗攻击承诺。
