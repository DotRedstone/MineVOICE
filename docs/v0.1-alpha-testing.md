# MineVOICE v0.1 Alpha 测试说明

欢迎参与 MineVOICE v0.1 Alpha 版本的测试！目前这是一个早期骨架版本，主要用于验证客户端-服务端架构、协议和基本的 3D 语音管线。

## 测试目标

1. **基本连接测试**：客户端是否能成功连接到 Standalone Server 的 UDP 端口并完成 `HELLO`/`AUTH` 握手。
2. **3D 音效测试**：在开启空间音频（Spatial Audio）时，声音是否会随着玩家距离的增加而衰减，以及是否能正确反映左右声道的声像偏移（Panning）。
3. **配置系统**：客户端能否通过设置界面正常修改输入/输出设备以及调整音量。

## 环境要求

- **Minecraft**: 1.21.1
- **NeoForge**: 21.1.233
- **Java**: 21 (无论是客户端还是独立服务端)

## 部署测试服务端

独立服务端可以通过直接运行 zip 发行版或 Docker 镜像启动。启动前请配置好 UDP 端口映射（默认 `25566`）并设置 `sharedSecret` 环境变量，具体参考 [Docker 部署说明](deployment.md) 或 [README](../README.md)。

## 注意事项

- 当前 Alpha 阶段已默认接入纯 Java Opus 编解码，但仍使用 Java Sound 采集/播放和基础空间化；真实多人听感、CPU 占用和带宽对比仍需继续验收，不推荐用于公开生产环境。
- 测试过程中如遇到意外崩溃或网络断开，请检查 `logs/latest.log` 和独立服务端的控制台输出。
- `pre-release-check` 脚本会自动在发版时校验防止关键 token 和意外产物被提交，保护测试期间的安全性。

## 提供反馈

发现任何错误或者可以改进的地方，请记录下客户端日志、服务端日志以及相关的重现步骤，并提交 Issue。感谢你的参与！
